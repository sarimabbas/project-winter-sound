We created in 37 user stories in total for the project. We further prioritised these user stories based on the user / project needs and the effort required. We prioritised 29 user stories as priority 1 which represent 76% of the total effort.

We took 8 user stories (worth 24 story points) in sprint 1 and completed all of them. It took us some time (first 4 days) to setup but we were able to accelerate in the last few days of the sprint

Sprint Completed Stories

  1. Be taken to a home page upon arrival on the web app.
  2. Access all events from the home page.
  3. See the time associated with each event.
  4. View more information about the web app via the about page.
  5. Always have the ability to navigate back to the home page via the footer
  6. Create a new event from the home page.
  7. Can obtain information about via rest api endpoints
  8. Can obtain all events via events rest api endpoint

        

Because we had an amazing team (there are 2 members of the team who are super-coders/ CSers), we were able to punch in a lot of progress during sprint 1. To get the home page started we were able to build a generic http server using golang, Allow golang to template html, host the app on AWS, and stand up ElephantSQL. We were also able to build most of the back end so that the coming sprints will be mostly front end development. To access the events we set up ORM in GoLang and build a home page route. So that users can obtain all events via events rest api endpoint, we queried all events from db, parse into json, and make http response with json.

Challenges Faced:

Getting started was the most challenging part. Initially we were unable to build and run the starter code locally. Our coder, Paul, created a new repo using Gin and Gorm that simplified some of the dependency management. This repo was published to an AWS EC2 instance.

Understanding how Gin passes context to the HTML templates and how to access that within the templates was also challenging. In particular, a quirk is that one value always has to be passed, otherwise the template is not rendered. Once we figured this out, the templating became quite easy. Sarim has prior experience with CSS and was able to create a flexbox layout with Bootstrap components.

Another challenge is componentization of our templates, which we are still figuring out. For example, we would not like to repeat header and footer. We will look to the initial starter code for guidance, but are also looking at the fresh new Web Components spec.
