Challenges faced/addressed:

In sprint 3 we were able to complete only 46% of the total remaining effort. This is because a significant portion (40%) of the stories were low priority and high effort stories, a large chunk of which we didn’t start as we weren’t able to complete a few priority 1 stories such as analyzing referral data and conducting A/B tests.

We faced the following challenges to finish the remaining 10% of the priority 1 stories:

  1. Analyzing referral traffic

Our original deployment was on AWS EC2, however that was not HTTPS enabled. Because of that we were not getting traffic referral sources on the Google Analytics dashboard. Analytics reports referral data as direct when a secure referrer (HTTPS) sends traffic to a non-secure website (HTTP).

Therefore, we switched our deployment to Heroku. The first-time setup was difficult and took time, since we had to create a new Heroku app, specify a Procfile and dependencies and set up automated deploy from our master branch. Once we start receiving traffic again to our new deploy, we will hopefully see the referral source. In the interim we are investigating how to redirect existing referrals from our old deploy to the new using a 301 redirect.

  1. Componentization

In an earlier sprint, we incurred some technical debt by not following the DRY principle (Don’t Repeat Yourself) in our HTML templates. This time around, we successfully modularized our code by using Go functions in our templates. Our templates are therefore much simpler. Our Analytics script, for instance, is part of a reusable <head> tag. Where we do need changes on a per-page basis (e.g. the page title), we simply pass it as an argument to our template functions. Going forward, adding new features or pages will require minimal repetition.

  1. A/B testing

We hoped to lay the groundwork for this in the current sprint, but are still investigating how to proceed. We are contending with two options: 1. Flipping a coin on the server-side and conditionally showing a different HTML template, or 2. Using Google Optimize (an Analytics product) that does not require modifying any code. The challenge with (1) is that the Professor’s test code might be looking for buttons with href=”/events/:id/donate” for both A and B, but we might need to change this for the B test to href=”/events/:id/support” in order for Analytics to track the difference in clicks. The challenge with (2) is that the modification is done client-side (using JavaScript). We are not sure if the Professor’s test code will be able to run Optimize’s code for Analytics to track. So we are currently in the process of investigating both approaches.
